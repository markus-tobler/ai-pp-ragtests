Evaluation design

One corpus, six ways to expose it

A curated, reproducible slice of MultiEURLEX with readable plain-text metadata, a 50-question test set spanning six capability tiers, and CompareMeaning as the single pass/fail signal.

300
English-language EU legal documents, each with a unique CELEX id
Regulation, Decision & Directive · 2012–2015 · ~2–10 pages each
5
Metadata dimensions for structured filtering
Policy domain, document type, year band, legal actor type, applicable role
50
Questions generated from the actual corpus
Every expected answer traces to a real source document by CELEX id

Question set — six capability tiers

TierCountPurpose
A9Broad topical questions with few metadata cues
B11Medium metadata constraints such as year and domain
C8Precise questions with 3+ constraints and exact facts
D7Tricky distractor cases where several documents look plausible
E5Unanswerable / control questions that require abstention
S10Semantic paraphrase questions with little or no lexical overlap

The six retrieval configurations

Scoring. Pass/fail uses CompareMeaning only — whether the agent answer means the same thing as the expected answer. GeneralQuality is reported for answer-form quality but is not the headline metric. Errors (empty responses or timeouts) appear in absolute totals but are excluded from the pass-rate denominator.

Discussion

Five findings from the data

The evaluation does not identify a single retrieval method that dominates every model. These observations follow directly from the per-run results above.

Finding 01

Retrieval and model interact

DV Knowledge is best on GPT‑4.1, MCP is best on GPT‑5 Chat, and Sonnet 4.6 makes several approaches look excellent. An architecture selected under one model can change rank under another.

Finding 02

Structured retrieval is the strongest tool-based default

MCP improves from 76.0% on GPT‑4.1 to 96.0% on GPT‑5 Chat and 100.0% over scored Sonnet cases — well matched to a corpus where year, type, domain, and act identifiers are decisive.

Finding 03

Knowledge granularity matters

The same corpus behaves very differently by shape. Per-document upload is the most reliable knowledge source; whole-corpus CSV and native table knowledge are more sensitive to model choice.

Finding 04

Semantic search is useful but not sufficient alone

Semantic is stable at 74.0%, 76.0%, and 90.0% but leads no run. Many hard questions need exact disambiguation among similar legal acts, which favours structured filters over similarity alone.

Finding 05

Errors reshape near-perfect scores

A method with 100.0% on 47 scored cases and three timeouts is not operationally equivalent to 100.0% on 50 scored cases. Pass rate and error counts must be read together.

Engineering guidance

Recommendations

For metadata-rich legal and policy corpora: use structured MCP retrieval as the default when tools are available; use the hybrid method when semantic recall is worth the added cost; for native knowledge grounding, prefer one uploaded file per document.

1

Default to MCP for metadata-rich Dataverse-backed corpora

It is the strongest GPT‑5 Chat method and reaches perfect scored accuracy under Sonnet 4.6. Its structured filters align with legal corpora where year, domain, document type, and act identifiers are decisive.

2

Use MCP plus Semantic when recall matters more than latency

A good robustness option when questions may be paraphrased or when missing a relevant document is costlier than running both the MCP and semantic-search paths.

3

For knowledge grounding, upload one file per document

DV Knowledge is the strongest knowledge-source option in every run. Avoid one large CSV as the primary knowledge object unless operational simplicity is worth the accuracy trade-off.

4

Treat native Dataverse table knowledge as a fallback

DV Table is last on both GPT runs and has errors under Sonnet 4.6. It may be convenient, but it should be evaluated carefully before production use.

5

Keep model selection in the retrieval decision

Sonnet 4.6 compresses the spread across methods, while GPT‑4.1 and GPT‑5 Chat expose larger differences. Re-run the evaluation when changing the model behind an agent.

6

Track failures and errors separately

CompareMeaning pass rate is the right headline retrieval metric, but timeout and empty-response counts should be reviewed alongside it for operational readiness.

7

Use the tiered question set to diagnose weaknesses

Tiers C and D expose exact legal disambiguation failures; Tier E checks abstention; Tier S checks paraphrase retrieval. The aggregate score alone hides which capability is failing.

Best scored result across the evaluation

DV Knowledge and MCP both reach a perfect CompareMeaning pass rate over their scored cases under Claude Sonnet 4.6 — read together with this run's three-error reliability caveat on four methods.
